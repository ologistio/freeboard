# Freeboard design system

This workshop is the living record of how Freeboard's interface works: the UX
philosophy and rules that govern it, and the brand assets and shared UI components
that implement them. Every component story renders the real Tailwind theme and
classes from `src/Freeboard/assets/css/app.css` - there is no separate copy to
keep in sync.

- **Foundations / UX philosophy** - the reasoning behind the interface.
- **Foundations / UX rules** - the numbered, testable form of that reasoning.
- **Brand / Colors** - the target colour palette (brand, neutrals, semantic) from the product prototype, shown for light and dark. Reference only; not yet wired into the app theme.
- **Brand / Typography** - the target type system: Schibsted Grotesk for human content, IBM Plex Mono for system marks, with the scale. Reference only.
- **Brand / Radii** - the corner-radius scale (2px seal to full pill), shown as a scale and in context. Reference only.
- **Brand / Elevation** - the elevation levels (flat, raised, drawer, modal) and the shadow values. Reference only.
- **Brand / Data marks** - how we present data: square status seals and dashed provenance stamps. Reference only; interactive component versions come later.
- **Components / Buttons** - the button component from the prototype (brand, default, quiet), with sizes and states. Reference only.
- **Components / Badges** - tags (soft-ground labels) and certification badges (mono, for external surfaces). Reference only.
- **Components / Cards & panels** - the panel container, the stat card, and the link card (with ghost add). Reference only.
- **Components / Form controls** - text fields, choices (checkbox, switch), and inline messaging. Reference only.
- **Components / Tables** - the workhorse list (grouped, exceptions-first), bulk selection, and the filter/search toolbar. Reference only.
- **Components / Tabs** - the underline-indicator tab bar (ARIA tablist) with switching panes, plus keyboard guidance. Reference only.
- **Components / Drawer** - the object-detail sheet: uniform anatomy (Anatomy) and the slide-in dialog over a scrim (Overlay). Reference only.
- **Components / Command palette** - the Ctrl-K entry (ARIA combobox/listbox): the palette anatomy and the interactive shortcut with keyboard guidance. Reference only.
- **Components / Nav rail** - the primary sidebar: brand, search entry, grouped nav with counts, active indicator, workspace switcher. Reference only.

- **Compositions / List page** - a full list page (header + actions, filter/search toolbar, table with row actions) assembled from the components, shown as the vendor directory.
- **Compositions / List with drawer** - a Controls list where clicking a control opens the detail drawer (the list -> drawer loop).
- **Compositions / Tabbed page** - the full Vendors page with Directory/Discovery/Reviews/Procurement as tabs.

Build components to satisfy the rules; cite rule numbers in review. Compositions assemble components into whole pages.

To add a component, add its class to `app.css` and a story under `stories/`.
Stories are plain HTML strings, matching the server-rendered Razor markup.
