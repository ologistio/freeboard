# Markdownlint

All Markdown files must pass markdownlint using the repo config
`.markdownlint.jsonc` (markdownlint defaults, line-length disabled, etc).

Fix issues rather than disabling rules. Check with
`npx markdownlint-cli2 "**/*.md"`.

## Carveout: OpenSpec generated and authored files

The OpenSpec CLI (`openspec init`/`openspec update`) generates instruction files
under `.claude/skills/openspec-*/` and `.claude/commands/opsx/`. These do not
follow the repo markdown rules and are regenerated by the CLI, so any hand-fix
would be overwritten.

OpenSpec change files under `openspec/changes/**` are also ignored. They use a
`## ...`-first heading convention (no top-level H1) that conflicts with MD041,
and they are authored and managed through the OpenSpec workflow rather than as
ordinary repo docs.

All three paths are ignored via `.markdownlint-cli2.jsonc` (`ignores`):
`.claude/skills/openspec-*/`, `.claude/commands/opsx/`, and
`openspec/changes/**`. Do not lint or hand-edit the CLI-generated files. Do not
widen the carveout to paths outside these OpenSpec directories.

See `CONTRIBUTING.md` for details.
