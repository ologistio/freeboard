# Conventional Commits

All commit messages must follow Conventional Commits v1.0.0.

Format: `<type>[optional scope]: <description>`, with an optional body and
footers. Common types: `feat`, `fix`, `build`, `chore`, `ci`, `docs`, `perf`,
`refactor`, `style`, `test`. Mark breaking changes with `!` before the colon and
a `BREAKING CHANGE:` footer.

See `CONTRIBUTING.md` for the full spec, scopes, and examples.
